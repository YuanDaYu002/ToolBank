简介：
LiteOS 不支持进程间通信机制，在移植live555时会比较麻烦，所以设备端只负责将数据帧通过P2P传输出去，
在linux主机上运行live555服务程序（经过改装，植入了P2P程序），向设备端发起P2P连接，申请拉流，然后将数据传给live555 rtsp服务程序，
之后可利用VLC播放器进行网络拉流，就可以实时观看视频。

1.该文件下:live/mediaServer/  库源码文件已经做过修改，针对嵌入式（linux_X64 / 君正T20 / 海思Hi3516EV100_liteos）移植
        A:live555MediaServer.cpp/.h     modified
        B:所有WW开头的文件              added
		C：P2P相关的源文件以及库文件等  added

2.live555最终生成的服务端程序是：live/mediaServer/live555MediaServer  ，只需要运行该文件就是启动服务端程序



编译步骤：
1.进入到live目录
2. #./genMakefiles linux
3. #make clean
4. #make
5.最终生成的服务端程序是：live/mediaServer/live555MediaServer
